# 소개

## 왜 Headless Chart인가?

차트를 만들 때 이런 고민을 해보신 적이 있나요?

- "이 차트 라이브러리는 기능은 좋은데, 우리 디자인 시스템에 맞지 않아..."
- "커스터마이징하려니 옵션이 너무 제한적이야"
- "처음부터 만들자니 시간이 너무 오래 걸릴 것 같고..."

**Headless Chart**는 바로 이런 문제를 해결하기 위해 만들어졌습니다.

## Headless가 뭔가요?

"Headless"는 UI 없이 로직만 제공한다는 의미입니다. Headless Chart는:

- ✅ **차트의 핵심 로직은 제공합니다** (데이터 계산, 스케일, 레이아웃)
- ✅ **디자인은 여러분이 결정합니다** (색상, 모양, 애니메이션)
- ✅ **완전한 커스터마이징이 가능합니다** (모든 컴포넌트를 교체 가능)

## Flitter 기반의 위젯 시스템

Headless Chart는 [Flitter](https://github.com/meursyphus/flitter) 프레임워크를 기반으로 만들어졌습니다. Flitter는 Flutter의 위젯 시스템을 웹에서 구현한 프레임워크로, 다음과 같은 특징을 가집니다:

- **선언적 UI**: 상태가 아닌 UI의 모습을 선언
- **컴포지션 기반**: 작은 위젯들을 조합해 복잡한 UI 구성
- **크로스 랜더러**: SVG와 Canvas 모두 지원

## 기존 차트 라이브러리와의 차이

### 일반적인 차트 라이브러리
```javascript
// 옵션으로 커스터마이징
new Chart({
  type: 'bar',
  options: {
    color: 'blue',
    borderWidth: 2,
    // 제한된 옵션들...
  }
})
```

### Headless Chart
```javascript
// 컴포넌트를 직접 교체
BarChart({
  data: myData,
  custom: {
    bar: MyCustomBar,      // 내가 만든 막대
    axis: MyCustomAxis,    // 내가 만든 축
    legend: MyCustomLegend // 내가 만든 범례
  }
})
```

## 주요 장점

### 1. 완전한 디자인 자유도
원하는 대로 모든 요소를 커스터마이징할 수 있습니다. 회사의 디자인 시스템에 완벽하게 맞출 수 있죠.

### 2. 점진적 커스터마이징
기본 제공되는 컴포넌트로 시작해서, 필요한 부분만 하나씩 교체할 수 있습니다.

### 3. 타입 안정성
TypeScript로 작성되어 완벽한 타입 지원을 제공합니다.

### 4. 프레임워크 독립적
React, Svelte, Vue 등 어떤 프레임워크에서도 사용할 수 있습니다.

## 이런 분들께 추천합니다

- 🎨 **디자이너와 협업하는 개발자**: 디자인 시안을 100% 구현하고 싶은 분
- 🏢 **디자인 시스템을 운영하는 팀**: 일관된 차트 컴포넌트가 필요한 분
- 🚀 **새로운 차트 타입이 필요한 분**: 기존 라이브러리에 없는 차트를 만들고 싶은 분
- 💡 **학습하고 싶은 분**: 차트가 어떻게 동작하는지 이해하고 싶은 분
